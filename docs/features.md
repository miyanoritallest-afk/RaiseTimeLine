## 3. ユースケース詳細

### UC-01: 新規ユーザー登録

| 項目 | 内容 |
|------|------|
| **概要** | 未登録ユーザーがメールアドレス・ユーザー名・パスワードを入力してアカウントを作成する |
| **事前条件** | ユーザーが未ログイン状態である |
| **基本フロー** | 1. 会員登録画面を開く<br>2. メールアドレス・ユーザー名・パスワード・パスワード確認を入力する<br>3. 「登録する」ボタンを押下する<br>4. 登録完了後、タイムライン画面へ遷移する |
| **代替フロー** | 3a. 入力値が不正な場合（メール形式エラー、パスワード不一致、重複メール）はエラーメッセージを表示して入力画面に留まる |
| **事後条件** | ユーザーアカウントが作成され、ログイン状態になる |

---

### UC-02: ログイン

| 項目 | 内容 |
|------|------|
| **概要** | 登録済みユーザーがメールアドレスとパスワードで認証し、アプリを利用できる状態にする |
| **事前条件** | ユーザーがアカウントを持ち、未ログイン状態である |
| **基本フロー** | 1. ログイン画面を開く<br>2. メールアドレスとパスワードを入力する<br>3. 「ログイン」ボタンを押下する<br>4. 認証成功後、タイムライン画面へ遷移する |
| **代替フロー** | 3a. 認証失敗（メールアドレスまたはパスワードが不正）の場合はエラーメッセージを表示する |
| **事後条件** | ログイン状態になり、認証が必要な全機能を利用できる |

---

### UC-03: ログアウト

| 項目 | 内容 |
|------|------|
| **概要** | ログイン中のユーザーがセッションを終了する |
| **事前条件** | ユーザーがログイン状態である |
| **基本フロー** | 1. ヘッダーのログアウトボタンを押下する<br>2. セッションが破棄される<br>3. ログイン画面へ遷移する |
| **代替フロー** | なし |
| **事後条件** | ログアウト状態になり、認証が必要な画面にアクセスできなくなる |

---

### UC-04: タイムライン閲覧（全体）

| 項目 | 内容 |
|------|------|
| **概要** | 全ユーザーの投稿を新着順で一覧表示する |
| **事前条件** | ユーザーがログイン状態である |
| **基本フロー** | 1. タイムライン画面を開く<br>2. 「おすすめ」タブを選択する<br>3. 全ユーザーの投稿が新着順に表示される<br>4. 各投稿にはアバター・ユーザー名・投稿内容・画像・いいね数・コメント数・投稿日時が表示される |
| **代替フロー** | 3a. 投稿が0件の場合は「まだ投稿がありません」を表示する |
| **事後条件** | 最新の投稿一覧が表示される |

---

### UC-05: タイムライン閲覧（フォロー中）

| 項目 | 内容 |
|------|------|
| **概要** | フォロー中のユーザーの投稿のみを新着順で表示する |
| **事前条件** | ユーザーがログイン状態である |
| **基本フロー** | 1. タイムライン画面を開く<br>2. 「フォロー中」タブを選択する<br>3. フォロー中ユーザーの投稿が新着順に表示される |
| **代替フロー** | 3a. フォロー中ユーザーが0人の場合は「フォロー中のユーザーがいません」を表示する<br>3b. フォロー中ユーザーの投稿が0件の場合は「まだ投稿がありません」を表示する |
| **事後条件** | フォロー中ユーザーの最新投稿一覧が表示される |

---

### UC-06: 投稿する

| 項目 | 内容 |
|------|------|
| **概要** | ユーザーがテキスト（最大280文字）と画像（最大4枚）を投稿する |
| **事前条件** | ユーザーがログイン状態である |
| **基本フロー** | 1. タイムライン画面の投稿フォームにテキストを入力する（最大280文字）<br>2. 任意で画像を最大4枚選択する<br>3. 「投稿する」ボタンを押下する<br>4. 画像はAWS S3にアップロードされる<br>5. 投稿がDBに保存され、タイムラインに反映される |
| **代替フロー** | 3a. テキストが空かつ画像が未選択の場合はエラーを表示する<br>3b. テキストが281文字以上の場合はエラーを表示する<br>3c. 画像が5枚以上選択された場合はエラーを表示する<br>3d. 画像アップロード失敗の場合はエラーを表示し投稿を中断する |
| **事後条件** | 投稿がDBに保存され、タイムラインの先頭に表示される |

---

### UC-07: 投稿を編集する

| 項目 | 内容 |
|------|------|
| **概要** | 自分の投稿のテキストを編集する |
| **事前条件** | ユーザーがログイン状態であり、編集対象の投稿が自分の投稿である |
| **基本フロー** | 1. 自分の投稿の「…」メニューから「編集」を選択する<br>2. テキストを編集する<br>3. 「保存する」ボタンを押下する<br>4. 投稿内容が更新される |
| **代替フロー** | 3a. テキストが空かつ画像が未選択の場合はエラーを表示する<br>3b. テキストが281文字以上の場合はエラーを表示する |
| **事後条件** | 投稿内容が更新され、タイムラインおよび詳細画面に反映される |

---

### UC-08: 投稿を削除する

| 項目 | 内容 |
|------|------|
| **概要** | 自分の投稿を削除する |
| **事前条件** | ユーザーがログイン状態であり、削除対象の投稿が自分の投稿である |
| **基本フロー** | 1. 自分の投稿の「…」メニューから「削除」を選択する<br>2. 確認ダイアログが表示される<br>3. 「削除する」ボタンを押下する<br>4. 投稿・関連画像（S3）・コメント・いいねが削除される |
| **代替フロー** | 3a. 「キャンセル」を押下した場合は削除せずダイアログを閉じる |
| **事後条件** | 投稿がDBおよびS3から削除され、タイムラインから消える |

---

### UC-09: コメントする

| 項目 | 内容 |
|------|------|
| **概要** | 任意の投稿にコメントを投稿する |
| **事前条件** | ユーザーがログイン状態である |
| **基本フロー** | 1. 投稿詳細画面を開く<br>2. コメント入力フォームにテキストを入力する<br>3. 「コメントする」ボタンを押下する<br>4. コメントがDBに保存され、コメント一覧に追加される<br>5. コメント数が+1される |
| **代替フロー** | 3a. テキストが空の場合はエラーを表示する |
| **事後条件** | コメントが保存され、投稿のコメント数が更新される |

---

### UC-10: コメントを編集・削除する

| 項目 | 内容 |
|------|------|
| **概要** | 自分のコメントを編集または削除する |
| **事前条件** | ユーザーがログイン状態であり、対象のコメントが自分のコメントである |
| **基本フロー（編集）** | 1. 自分のコメントの「…」メニューから「編集」を選択する<br>2. テキストを編集する<br>3. 「保存する」ボタンを押下する<br>4. コメント内容が更新される |
| **基本フロー（削除）** | 1. 自分のコメントの「…」メニューから「削除」を選択する<br>2. 確認ダイアログを確認する<br>3. 「削除する」を押下する<br>4. コメントが削除され、コメント数が-1される |
| **代替フロー** | 3a. 編集時にテキストが空の場合はエラーを表示する |
| **事後条件** | コメントが更新または削除され、コメント数が反映される |

---

### UC-11: いいねする／取り消す

| 項目 | 内容 |
|------|------|
| **概要** | 任意の投稿にいいねをする、またはいいねを取り消す |
| **事前条件** | ユーザーがログイン状態である |
| **基本フロー（いいね）** | 1. タイムラインまたは投稿詳細画面でいいねボタン（♡）を押下する<br>2. いいねがDBに保存される<br>3. いいねボタンが塗りつぶされ（♥）、いいね数が+1される |
| **基本フロー（取り消し）** | 1. いいね済みの投稿のいいねボタン（♥）を押下する<br>2. いいねがDBから削除される<br>3. いいねボタンが空（♡）に戻り、いいね数が-1される |
| **代替フロー** | 1a. 自分の投稿にもいいねできる |
| **事後条件** | いいね状態といいね数がDBに反映される |

---

### UC-12: ユーザーをフォローする

| 項目 | 内容 |
|------|------|
| **概要** | 任意のユーザーをフォローする、またはフォローを解除する |
| **事前条件** | ユーザーがログイン状態である |
| **基本フロー（フォロー）** | 1. プロフィール画面または検索結果でフォローボタンを押下する<br>2. フォロー関係がDBに保存される<br>3. ボタンが「フォロー解除」に変わる<br>4. フォロー数・フォロワー数が更新される |
| **基本フロー（フォロー解除）** | 1. 「フォロー解除」ボタンを押下する<br>2. フォロー関係がDBから削除される<br>3. ボタンが「フォローする」に変わる |
| **代替フロー** | 1a. 自分自身はフォローできない（ボタンを非表示にする） |
| **事後条件** | フォロー関係がDBに反映され、フォロー中タイムラインに影響する |

---

### UC-13: ユーザーを検索する

| 項目 | 内容 |
|------|------|
| **概要** | ユーザー名（部分一致）でユーザーを検索し、フォロー状態を確認する |
| **事前条件** | ユーザーがログイン状態である |
| **基本フロー** | 1. ヘッダーの検索アイコンまたは検索画面を開く<br>2. 検索フォームにユーザー名の一部を入力する<br>3. 検索結果にユーザーカード（アバター・ユーザー名・フォロー状態）が表示される<br>4. 検索結果からフォローボタンを押下してフォローできる<br>5. ユーザーカードをクリックするとプロフィール画面へ遷移する |
| **代替フロー** | 3a. 検索結果が0件の場合は「ユーザーが見つかりませんでした」を表示する |
| **事後条件** | 検索結果が表示される |

---

### UC-14: プロフィールを確認する

| 項目 | 内容 |
|------|------|
| **概要** | 任意のユーザーのプロフィール情報と投稿一覧を確認する |
| **事前条件** | ユーザーがログイン状態である |
| **基本フロー** | 1. 投稿やコメントのユーザー名をクリック、またはプロフィール画面URLに直接アクセスする<br>2. アバター画像・ユーザー名・自己紹介・フォロー数・フォロワー数が表示される<br>3. そのユーザーの投稿一覧が新着順に表示される<br>4. 他ユーザーのプロフィールの場合はフォローボタンが表示される |
| **代替フロー** | 1a. 存在しないユーザーIDにアクセスした場合は404ページを表示する |
| **事後条件** | 対象ユーザーのプロフィールと投稿一覧が表示される |

---

### UC-15: プロフィールを編集する

| 項目 | 内容 |
|------|------|
| **概要** | 自分のプロフィール（アバター画像・ユーザー名・自己紹介）を編集する |
| **事前条件** | ユーザーがログイン状態であり、自分のプロフィール画面を表示している |
| **基本フロー** | 1. 自分のプロフィール画面の「プロフィール編集」ボタンを押下する<br>2. プロフィール編集画面でアバター画像・ユーザー名・自己紹介を編集する<br>3. アバター画像を変更する場合はAWS S3にアップロードされる<br>4. 「保存する」ボタンを押下する<br>5. プロフィール画面に戻り、更新内容が反映される |
| **代替フロー** | 4a. ユーザー名が空の場合はエラーを表示する<br>4b. 画像アップロード失敗の場合はエラーを表示する |
| **事後条件** | プロフィール情報がDBに更新され、各画面のアバターとユーザー名に反映される |
