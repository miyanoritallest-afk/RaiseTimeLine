# RaiseTimeLine インフラ構成

作成日: 2026-05-12
最終更新日: 2026-05-12

> **注意**: AWS上でのサーバー構築は現時点で決定事項ではありません。本ドキュメントはEC2/RDS/ALBを用いた構成を前提とした設計案です。

---

## 1. 概要

| 項目 | 内容 |
|------|------|
| クラウドプロバイダー | AWS |
| 画像ストレージ | AWS S3（決定済み） |
| アプリケーションサーバー | AWS EC2（構築予定） |
| データベースサーバー | AWS RDS for PostgreSQL（構築予定） |
| ロードバランサー | AWS ALB（Application Load Balancer）（構築予定） |
| ローカル開発DB | Docker Compose (PostgreSQL) |

## 3. AWSリソース一覧

| リソース | サービス | 役割 | 備考 |
|---------|---------|------|------|
| ALB | Application Load Balancer | HTTPトラフィックの受付・EC2への振り分け | ポート80/443 |
| EC2 | Amazon EC2 | アプリケーションサーバー | Spring Boot + Nginx |
| RDS | Amazon RDS for PostgreSQL | データベース | プライベートサブネット配置 |
| S3 | Amazon S3 | 画像ファイルの永続化 | パブリックアクセス制御 |
| VPC | Amazon VPC | ネットワーク分離 | パブリック/プライベートサブネット |
| Security Group | EC2/RDS用 | ネットワークアクセス制御 | 下記参照 |

## 4. ネットワーク・セキュリティ

### EC2 セキュリティグループ

| ポート | プロトコル | 許可元 | 用途 |
|--------|----------|--------|------|
| 80 | TCP | ALBセキュリティグループ | HTTPリクエスト受付 |
| 8080 | TCP | ALBセキュリティグループ | Spring Boot APIへの転送 |
| 22 | TCP | 管理者IP | SSH接続（メンテナンス用） |

### RDS セキュリティグループ

| ポート | プロトコル | 許可元 | 用途 |
|--------|----------|--------|------|
| 5432 | TCP | EC2セキュリティグループのみ | PostgreSQL接続 |

### ALB セキュリティグループ

| ポート | プロトコル | 許可元 | 用途 |
|--------|----------|--------|------|
| 80 | TCP | 0.0.0.0/0（インターネット） | HTTPアクセス受付 |
| 443 | TCP | 0.0.0.0/0（インターネット） | HTTPSアクセス受付（将来対応） |

## 5. EC2 上のソフトウェア構成

| ソフトウェア | 役割 | 起動方法 |
|------------|------|---------|
| Java 21 | Spring Boot 実行環境 | システム起動時に自動起動 |
| Spring Boot JAR | REST API サーバー（ポート8080） | systemd サービスとして起動 |
| Nginx | 静的ファイルホスティング + リバースプロキシ | systemd サービスとして起動 |

**Nginx 設定方針:**
- `/` → React ビルド済み静的ファイルを返す
- `/api/` → `localhost:8080` へプロキシ

## 6. S3 バケット設計

| バケット | 用途 | アクセス制御 |
|---------|------|------------|
| `raisetimeline-images` | 投稿画像・アバター画像 | EC2からのみアップロード可、パブリック読み取り可（画像表示のため） |

**ディレクトリ構成（S3オブジェクトキー）:**

```
raisetimeline-images/
├── avatars/
│   └── {user_id}/{filename}
└── posts/
    └── {post_id}/{display_order}_{filename}
```

## 7. ローカル開発環境との対応

| 本番環境 | ローカル開発環境 |
|---------|---------------|
| ALB + EC2 | localhost（ポート8080 / 5173） |
| RDS (PostgreSQL) | Docker Compose (PostgreSQL) |
| S3 | S3（ローカルではS3に直接接続、またはLocalStackで代替可） |

---

## 8. 注意事項

- RDSはプライベートサブネットに配置し、インターネットから直接アクセスできない構成とする
- EC2のSSH接続は管理者IPのみに限定する
- S3バケットポリシーで画像の公開範囲を適切に制限する
- AWSの認証情報（アクセスキー等）はEC2のIAMロールで管理し、コードに埋め込まない
