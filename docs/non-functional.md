# RaiseTimeLine 非機能要件

作成日: 2026-05-12
最終更新日: 2026-05-12

---

## 1. パフォーマンス

| 項目 | 要件 |
|------|------|
| タイムライン初期表示 | 2秒以内（通常のネットワーク環境） |
| API レスポンス | 500ms以内（DBクエリを含む） |
| 画像アップロード | ユーザーへのフィードバックを表示しつつ非同期で処理する |
| タイムライン取得件数 | 1回あたり最大20件（ページネーション対応） |
| 画像1枚あたりのサイズ上限 | 5MB |

---

## 2. セキュリティ

| 項目 | 要件 |
|------|------|
| 認証方式 | JWT（JSON Web Token）、有効期限あり |
| パスワード保存 | BCryptによるハッシュ化（平文保存禁止） |
| 入力バリデーション | フロントエンド・バックエンドの両方でバリデーションを実施 |
| XSS対策 | Reactのデフォルトエスケープに準拠、dangerouslySetInnerHTMLは使用しない |
| CSRF対策 | JWT認証によりCookieベースのセッションを使用しないため対象外 |
| APIエンドポイント保護 | 認証が必要なエンドポイントはJWTなしのリクエストを拒否（401を返す） |
| 権限チェック | 他ユーザーの投稿・コメントの編集・削除はバックエンドで拒否（403を返す） |
| S3アクセス | IAMロールを使用し、アクセスキーのコード埋め込みを禁止する |

---

## 3. 対応ブラウザ・デバイス

| 項目 | 要件 |
|------|------|
| 対応ブラウザ | Google Chrome（最新版）、Microsoft Edge（最新版）、Firefox（最新版）、Safari（最新版） |
| 対応デバイス | PC（デスクトップ・ラップトップ）を主対象とする |
| 推奨画面幅 | 1024px 以上 |
| モバイル対応 | スコープ外（将来の拡張候補） |

---

## 4. データ整合性

| 項目 | 要件 |
|------|------|
| DBマイグレーション | Flywayによりスキーマ変更を管理する |
| 投稿削除時のカスケード | 投稿削除時に関連するコメント・いいね・添付画像（S3含む）を連動削除する |
| いいねの重複防止 | DBのUNIQUE制約 `(post_id, user_id)` で重複いいねを防ぐ |
| フォローの重複防止 | DBのUNIQUE制約 `(follower_id, following_id)` で重複フォローを防ぐ |
| 自己フォロー禁止 | DBのCHECK制約 `follower_id <> following_id` で自己フォローを防ぐ |

---

## 5. 可用性・運用（本番環境）

| 項目 | 要件 |
|------|------|
| ロードバランサー | ALBにより単一障害点を排除する（将来的なスケールアウト対応） |
| DB可用性 | RDSの自動バックアップを有効化する |
| ログ | Spring Boot のアプリケーションログをサーバーに保存する |

---

## 6. アクセシビリティ

| 項目 | 要件 |
|------|------|
| キーボード操作 | 主要な操作（ログイン・投稿・いいね）はキーボードで実行できる |
| フォーカス管理 | フォーカスが視覚的に判別できるようにする |
| 色コントラスト | WCAG 2.1 AA 基準（コントラスト比 4.5:1 以上）を目標とする |
